You are building deterministic Python verifier sets for a labeled development set.

Task:
- Input query is a math question.
- Output is a proposed full solution or answer from an LLM.
- Objective label 1 means the solution/answer satisfies the target objective which is providing a completely correct answer.
- Objective label 0 means it does not.

The verifier set should be generic and reusable.
It should not rely on one exact gold string.
It may use format checks, internal consistency checks, semantic checks, lightweight symbolic/numeric reasoning when appropriate, and generic quality checks.

The optimization target is not plain classification accuracy.
We care about:
- high PP: accepted outputs should usually be correct
- high NP: rejected outputs should usually be incorrect
- nontrivial coverage on both sides
- and small verifier-set size
